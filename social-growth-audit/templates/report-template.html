04 / X 作战策略

X：把火力全部交给长文

4.1 主页改造 — Day 1 完成，单次最高 ROI

点进主页的人 3 秒内要能回答「关注他我能持续得到什么」。现在的 bio（AILab/yes工程师 • Vibecoding • AI Agent • Workflow）是标签堆砌，没有承诺。

Bio 公式 = 身份钩子 + 关注承诺 + 信任凭证

示例：把想法用 Claude Code 变成产品的人（非科班，所以教程你一定看得懂）
　　　｜每周三 / 日更新 AI 工作流教程 + 可白嫖小工具｜开源 Styledex ⭐

「非科班」不是短板是卖点——你的目标读者就是非科班的 AI 工具使用者。「我和你一样不是程序员」比工程师人设可信度更高，且永远不会露馅。

  • 置顶帖更换：现在置顶的 Styledex 演示已被证伪（2.3K 曝光 5 赞）。换成「内容索引 thread」——首条写关注理由，后续挂 2 篇爆款长文 + Styledex + 教程预告；每发新长文就更新进去，让置顶帖成为活目录。
  • 停止关注新账号（互关除外），让 493:325 不再恶化。
  • 时间线两周保持纯净：搞钱感想、吐槽、生活照暂停。点进主页的人要看到连续、同质、可期待的输出。

4.3 长文 CTA 三件套 — 把收藏党转成关注党

5/30 那篇结尾写了「点个关注」，222 个收藏者只有 6 个照做。修正：

  1. 预告下一篇：「关注我，下周三更新《具体标题》」——把关注从客套变成追更动作。
  2. 中段插一次软 CTA：读到结尾才看到 CTA 的人，已经在收藏离开的路上了。
  3. 评论区钩子：发文 1 小时内自己置顶评论——「本文配套 prompt 打包：关注后私信『工作流』自取」。

4.4 选题库 — 半成品恰好够用

过程文比成品文更适合现在的你，build in public 的本质就是卖过程。两周 4 篇，全部绑定 Fable 5 热点关键词，吃搜索 + 话题双流量：

  • 《非程序员用 Claude Code 的第一周：我踩的 5 个坑和 3 个开窍瞬间》— 新手向，「非科班」即差异化
  • 《手把手：给你的项目写第一份 CLAUDE.md / AGENTS.md（附可抄模板）》— 5/30 爆文的直接衍生，需求已验证
  • 《我用 Fable 5 重构了 Styledex：半成品也敢 build in public 的工作流》— 产品 + 教程杂交，给 Styledex 二次曝光
  • 《写给想入坑 vibecoding 的普通人：从 0 到第一个能用的工具》— 面向最大基数的新手盘

  • AIGC 内容按平台要求主动标注，教程图文用自己的真实截图最安全。

5.2 涨粉笔记公式

小红书涨粉的核心机制 = 系列化。关注的动机不是「这篇好」，而是「我怕错过下一篇」。

封面　　= 大字报风格（你已会）+ 系列角标「AI 普通人逆袭 Day N」
标题　　= 痛点开头（"还在手打提示词？" / "AI 做的网页丑哭了？"）
正文图　= 3–6 张步骤图，每张一个动作，小白能照抄
末图　　= 固定版式「下一篇：《具体标题》｜关注不迷路」
文案　　= ≤300 字口语化，结尾提问引导评论
置顶评论= 补充干货（"完整 prompt 在这"）+ 引导关注

08 / 风险与纪律

不变形，才能复利

  1. 不买粉、不刷量。你的下游目标（小红书商单、X 影响力）都依赖真实画像，假粉是负资产。
  2. 互关控制在新增 30% 以内，且只互关同赛道账号。
  3. 小红书合规：不站外导流、产品笔记用分享体、AIGC 按要求标注。商单需 ≥1000 粉开通蒲公英，本阶段不做变现动作。
  4. 人设安全区：「不是真程序员」反着用——所有教程从非科班视角写，既是差异化也是免露馅保险。唯一雷区：不写底层原理的断言式内容，遇到用「我的理解是…欢迎指正」句式。
  5. 两周后无论结果，回到指标表复盘再定下一周期。涨粉是复利游戏，单周期赌输赢必然动作变形。
